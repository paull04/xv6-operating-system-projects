**COW** 구현

**COW 분석**  
Copy-On-Write(COW)는 페이지를 공유하는 방식으로 메모리 사용을 최적화하는 기법이다. 페이지가 공유되다가 쓰기 작업이 발생하면, 해당 페이지를 복사하여 새로운 페이지를 할당하고, 쓰기 작업은 새 페이지에 적용된다. 이를 통해 불필요한 메모리 복사를 방지할 수 있다. 프로세스 포크가 발생하면 부모 프로세스의 페이지 테이블을 복사하여 자식 테이블의 페이지 테이블로 사용한다. 이 때 PTE_W 비트를 끄고 PTE_COW 비트를 켜서 COW로 페이지가 공유돼 있음을 OS에 알려준다. 이후 해당 페이지를 수정하려고 하면 PTE_W 비트가 켜져있기 때문에 페이지 폴트가 발생하고 핸들러를 통해 페이지를 할당하고 값을 복사하여 해당 페이지에 쓰기 작업을 수행한다. 이 과정에서는 PTE_W 비트를 켜준다. COW는 유저 페이지에 대해서만 적용되기 때문에 실제 구현에서는 유저 메모리를 다루는 함수에 대해서만 수정이 필요하다.

**uvmcopy** 분석
```C

기존의 uvmcopy는 모든 page에 대해 새롭게 물리주소를 할당(kalloc)하고 memmove로 값을 복사햇다. COW에서는 이를 물리주소를 공유하게 변경해야 한다. uvmcopy는 유저 페이지에 대해서만 동작하기 때문에 따로 유저 페이지 flag를 확인할 필요는 없다.

**COW 구현**

**reference counting**  
총 PAGE의 개수는 전체 주소 / PAGE_SIZE로 구할 수 있다. 페이지 개수만큼의 길이를 갖는 배열을 만들어 각 페이지의 reference count를 저장한다.  
```C
#define N_PHY_PAGES ((PHYSTOP) >> PGSHIFT)
static int pgref[N_PHY_PAGES];
```

그리고 reference count를 증가시키는 함수와 감소시키는 함수, 그리고 reference count를 반환하는 함수를 구현한다. 그리고 reference count에 접근할 때는 pgref_lock을 사용해 동기화한다. decrement_ref 함수에서 reference count가 0이 되면 해당 페이지를 해제한다.  
```C
void increment_ref(uint64 pa){
  if (pa >= PHYSTOP) {
    panic("increment_ref: invalid pa");
  }
  acquire(&pgref_lock);
  ++pgref[pa >> PGSHIFT];
  //printf("inc: pgref[%lu]=%d\n", pa, r);
  release(&pgref_lock);
}
void decrement_ref(uint64 pa){
  if (pa >= PHYSTOP) {
    panic("decrement_ref: invalid pa");
  }
  acquire(&pgref_lock);
  int r = --pgref[pa >> PGSHIFT];
  //printf("dec: pgref[%lu]=%d\n", pa, r);
  release(&pgref_lock);
  if(r == 0) kfree((void *)pa);
}
int get_ref(uint64 pa){
  if (pa >= PHYSTOP) {
    panic("decrement_ref: invalid pa");
  }
  acquire(&pgref_lock);
  int r = pgref[pa >> PGSHIFT];
  release(&pgref_lock);
  return r;
}
```

**page_change** 함수 구현
```C
uint64 page_change(pagetable_t pagetable, pte_t *pte, uint64 va0){
  uint64 oldpa = PTE2PA(*pte);
  uint64 pa = oldpa;
  if(get_ref(oldpa) > 1){
    pa = (uint64)kalloc();
    if(pa == 0) return 0;
    memmove((void*)pa, (void*)oldpa, PGSIZE);
    decrement_ref(oldpa);
    increment_ref(pa);
  }
  uint64 flags = PTE_FLAGS(*pte);
  flags |= PTE_W;
  flags &= ~PTE_COW;
  *pte = PA2PTE(pa) | flags;
  sfence_vma();
  return pa;
}
```
page_change 함수는 페이지가 공유되고 있는 경우 새롭게 페이지를 할당하고 값을 복사하여 해당 페이지에 쓰기 작업을 수행한다. 만약 페이지가 공유되고 있지 않다면 기존의 물리 주소를 그대로 사용한다. 페이지가 공유되고 있는지는 reference count로 확인할 수 있다. 그리고 PTE_W 비트를 켜고 PTE_COW 비트를 끈다. pte가 수정됐으니 sfence_vma를 호출하여 TLB를 갱신한다.

**re_page** 함수 구현
```C
uint64 re_page(pagetable_t pagetable, pte_t *pte, uint64 va0){
  return (*pte & PTE_W)?
    PTE2PA(*pte) :
    page_change(pagetable, pte, va0);
}
re_page는 PTE_W 비트가 켜져있으면 기존의 물리 주소를 그대로 사용하고, PTE_W 비트가 꺼져있고 PTE_COW 비트가 켜져있으면 page_change 함수를 호출하여 페이지를 새롭게 할당하고 해당 물리주소를 반환한다.


**pagefault_handler** 구현
```C
int
pagefault_handler(pagetable_t pagetable, uint64 va){
  pte_t *pte;
  va = PGROUNDDOWN(va);
  if ((pte = walk(pagetable, va, 0)) == 0 || (*pte & PTE_V) == 0) return -1;
  else if((*pte & PTE_COW) == 0) return -1;
  if(page_change(pagetable, pte, va) == 0) return -1;
  return 0;
}
```
pagefault_handler 함수는 페이지 폴트가 발생했을 때 호출된다. 이 함수는 페이지 테이블에서 해당 가상 주소의 pte를 찾고, PTE_COW 비트가 켜져있는지 확인한다. PTE_COW 비트가 켜져있으면 page_change 함수를 호출하여 페이지를 새롭게 할당하고, 그렇지 않으면 -1을 반환한다.

**Doubly Indirect Block** 구현


**Doubly Indirect Block 분석**  
파일 시스템에서 Block Addressing의 구조는 direct 12개 indirect 1개로 이뤄져있다. direct 주소는 data로 직접연결되는 주소를 갖고 있고 indirect 주소는 address 256개를 가르키는 블록을 가르키고 있다. 여기서 추가해야 하는 것은 Doubly Indirect 구조로 Indirect 블록 256개를 갖고 있는 블록을 가르키는 주소를 추가해야 한다.

**코드 분석**

**dinode 및 inode** 분석
```C
struct dinode {
  ...
  uint addrs[NDIRECT+1]; // Data block addresses (NDIRECT direct, 1 indirect)

struct inode {
  ...
  uint addrs[NDIRECT+1];
};
};
```
해당 구조체에서 핵심부분은 addrs이다. addrs는 direct 주소 12개와 indirect 주소 1개를 갖고 있다. doubly indirect block을 추가하기 위해서는 addrs를 1개 더 늘려서 총 2개의 indirect 주소를 갖도록 해야 한다.

**bmap** 분석
```C
static uint
bmap(struct inode *ip, uint bn)
{
  uint addr, *a;
  struct buf *bp;

  if(bn < NDIRECT){
    if((addr = ip->addrs[bn]) == 0){
      addr = balloc(ip->dev);
      if(addr == 0)
        return 0;
      ip->addrs[bn] = addr;
    }
    return addr;
  }
  bn -= NDIRECT;

  if(bn < NINDIRECT){
    // Load indirect block, allocating if necessary.
    if((addr = ip->addrs[NDIRECT]) == 0){
      addr = balloc(ip->dev);
      if(addr == 0)
        return 0;
      ip->addrs[NDIRECT] = addr;
    }
    bp = bread(ip->dev, addr);
    a = (uint*)bp->data;
    if((addr = a[bn]) == 0){
      addr = balloc(ip->dev);
      if(addr){
        a[bn] = addr;
        log_write(bp);
      }
    }
    brelse(bp);
    return addr;
  } 
  panic("bmap: out of range");
}
```
bmap은 block number를 실제 disk 주소에 매핑하는 함수이다. Direct에 대해 매핑돼 있고 주소가 할당돼있지 않으면 balloc함수를 사용해 할당한다. Indirect로 매핑될 때는 data 주소들을 저장하기 위한 block을 할당하고 bread를 통해 해당 block을 읽어온다 block에서 bn에 해당하는 주소가 할당돼 있지 않으면 balloc을 통해 할당하고 log_write를 통해 해당 block을 기록한다. 그리고 bread로 읽어온 block은 brelse를 통해 해제한다. doubly indirect block을 추가하려면 Indirect로 매핑되는 부분의 코드를 사용해 코드를 추가해야 한다. Indirect block을 할당하는 과정은 balloc을 통해 할당하고 해당 블록에 데이터 주소를 저장하는 과정이었다. Doubly indirect block은 balloc을 통해 indiret block을 두번 할당한다. 


**itrunc** 분석
```C
void
itrunc(struct inode *ip)
{
  int i, j, k;
  struct buf *bp, *bp2;
  uint *a, *b;

  for(i = 0; i < NDIRECT; i++){
    if(ip->addrs[i]){
      bfree(ip->dev, ip->addrs[i]);
      ip->addrs[i] = 0;
    }
  }

  if(ip->addrs[NDIRECT]){
    bp = bread(ip->dev, ip->addrs[NDIRECT]);
    a = (uint*)bp->data;
    for(j = 0; j < NINDIRECT; j++){
      if(a[j])
        bfree(ip->dev, a[j]);
    }
    brelse(bp);
    bfree(ip->dev, ip->addrs[NDIRECT]);
    ip->addrs[NDIRECT] = 0;
  }
  ip->size = 0;
  iupdate(ip);
}
```
itrunc는 inode의 데이터를 모두 해제하는 함수이다. Direct 주소에 대해 bfree를 호출하여 해당 주소를 해제하고, Indirect 주소에 대해서도 bread를 통해 해당 block을 읽어온 후, 해당 block의 주소들을 bfree를 통해 해제한다. 그리고 Indirect 주소를 해제하고 inode의 크기를 0으로 설정한 후 iupdate를 호출하여 inode를 업데이트한다. doubly indirect block을 추가하려면 indirect block 두 번거쳐서 해제해야 하기 때문에 Indirect 주소를 해제하는 부분을 두 번 추가해야 한다. 이는 이중 for문을 사용할 것이다.

static uint
bmap(struct inode *ip, uint bn)
{
  bn -= NINDIRECT;
  if(bn < NDINDIRECT){
    if((addr = ip->addrs[NIRECT]) == 0){
      addr = balloc(ip->dev);
      if(addr == 0)
        return 0;
      ip->addrs[NIRECT] = addr;
    }
    bp = bread(ip->dev, addr);
    a = (uint*)bp->data;
    uint i1 = bn >> 8;
    uint i2 = bn & 0xff;
    if((addr = a[i1]) == 0){
      addr = balloc(ip->dev);
      if(addr == 0) return 0;
      a[i1] = addr;
      log_write(bp);
    }
    brelse(bp);
    bp = bread(ip->dev, addr);
    a = (uint*)bp->data;
    if((addr = a[i2]) == 0){
      addr = balloc(ip->dev);
      if(addr){
        a[i2] = addr;
        log_write(bp);
      }
    }
    brelse(bp);
    return addr;
  }
  panic("bmap: out of range");
}
```
bn이 Double indirect block에 할당되는 경우 첫번째 block과 두번째 block에 대한 인덱스가 필요하다. 첫번째 block은 256개의 주소를 갖고 있고, 두번째 block은 256개의 주소를 갖고 있다. 따라서 bn을 256으로 나눈 몫을 i1로, 나머지를 i2로 설정한다. 그리고 첫번째 block에 대한 주소를 할당하고, 해당 주소를 bread를 통해 읽어온 후, 두번째 block에 대한 주소를 할당한다. 두번째 블록의 a[i2]는 data에 대해서 할당이 이뤄진다.

**itrunc** 수정
```C
void
itrunc(struct inode *ip)
{
  ...
  if(ip->addrs[NIRECT]){
    bp = bread(ip->dev, ip->addrs[NIRECT]);
    a = (uint*)bp->data;
    for(j = 0; j < NINDIRECT; j++){
      if(a[j]){
        bp2 = bread(ip->dev, a[j]);
        b = (uint*)bp2->data;
        for(k = 0; k < NINDIRECT; k++){
          if(b[k]){
            bfree(ip->dev, b[k]);
          }
        }
        brelse(bp2);
        bfree(ip->dev, a[j]);
      }
    }
    brelse(bp);
    bfree(ip->dev, ip->addrs[NIRECT]);
    ip->addrs[NIRECT] = 0;
  }

  ip->size = 0;
  iupdate(ip);
}
```
doubly indirect block을 해제하는 코드를 추가했다. 이중 for문으로 각각 256까지 반복한다. 첫번째 for문에서는 주소를 담고 있는 block을 두 번째 for문에서는 데이터를 가르키는 block을 가져오고 해당 주소가 할당돼 있으면 bfree를 호출하여 해당 주소를 해제한다. 그리고 두 번째 for문이 끝나면 첫번째 for문에서 접근한 block을 해제하고, 첫 번째 for문이 끝나면 doubly indirect block을 해제한다.

**symbolic link 구현**

**Symbolic Link 분석**  
Symbolic Link는 파일 시스템에서 파일의 경로를 가리키는 특수한 파일이다. 일반적인 파일은 데이터를 직접 저장하지만, 심볼릭 링크는 다른 파일이나 디렉토리의 경로를 저장한다. 심볼릭 링크를 사용하면 파일 시스템 내에서 다른 위치에 있는 파일을 참조할 수 있다. 심볼릭 링크는 일반적으로 상대 경로나 절대 경로를 사용하여 대상 파일을 가리킨다. 심볼릭 링크는 원본 파일이 삭제되거나 이동되더라도 링크가 깨지지 않는 한 계속 사용할 수 있다.

**sys_open** 분석
```C
uint64
sys_open(void)
{
  char path[MAXPATH];
  int fd, omode;
  struct file *f;
  struct inode *ip;
  int n;

  argint(1, &omode);
  if((n = argstr(0, path, MAXPATH)) < 0)
    return -1;

  begin_op();

  if(omode & O_CREATE){
    ip = create(path, T_FILE, 0, 0);
    if(ip == 0){
      end_op();
      return -1;
    }
  }
  else {
    if((ip = namei(path)) == 0){
      end_op();
      return -1;
    }
    ilock(ip);
    if(ip->type == T_DIR && omode != O_RDONLY){
      iunlockput(ip);
      end_op();
      return -1;
    }
  }

  if(ip->type == T_DEVICE && (ip->major < 0 || ip->major >= NDEV)){
    iunlockput(ip);
    end_op();
    return -1;
  }

  if((f = filealloc()) == 0 || (fd = fdalloc(f)) < 0){
    if(f)
      fileclose(f);
    iunlockput(ip);
    end_op();
    return -1;
  }

  if(ip->type == T_DEVICE){
    f->type = FD_DEVICE;
    f->major = ip->major;
  } else {
    f->type = FD_INODE;
    f->off = 0;
  }
  f->ip = ip;
  f->readable = !(omode & O_WRONLY);
  f->writable = (omode & O_WRONLY) || (omode & O_RDWR);

  if((omode & O_TRUNC) && ip->type == T_FILE){
    itrunc(ip);
  }

  iunlock(ip);
  end_op();

  return fd;
}
```
sys_open 함수는 파일을 열 때 호출되는 시스템 콜이다. 이 함수는 파일 경로와 모드를 인자로 받아서 해당 파일을 열고 파일 디스크립터를 반환한다. 만약 O_CREATE 플래그가 설정되어 있으면 create 함수를 호출하여 새로운 파일을 생성하고, 그렇지 않으면 namei 함수를 호출하여 해당 경로의 inode를 찾는다. 만약 O_NOFOLLOW 플래그가 설정되어 있지 않고, 파일이 심볼릭 링크인 경우, 심볼릭 링크의 내용을 읽어서 실제 파일 경로를 찾는다. 이 과정은 심볼릭 링크가 여러 번 중첩될 수 있기 때문에 최대 10번까지 반복한다. 만약 심볼릭 링크가 계속해서 발견되면 -1을 반환하고 종료한다. 이후 파일 디스크립터를 할당하고, 파일 타입에 따라 설정을 한다. 만약 O_TRUNC 플래그가 설정되어 있고, 파일 타입이 T_FILE인 경우 itrunc 함수를 호출하여 해당 파일의 내용을 비운다.

symlink(char *target, char *path){
  struct inode *ip;
  int n1 = strlen(target);
  begin_op();
  if((ip = create(path, T_SYMLINK, 0, 0)) == 0){
    end_op();
    return -1;
  }
  int r = writei(ip, 0, (uint64)target, 0, n1 + 1);
  if(r != n1 + 1) {
  ip->nlink = 0;
    iupdate(ip);
    iunlockput(ip);
    end_op();
    return -1;
  }
  iupdate(ip);
  iunlockput(ip);

  end_op();
  return 0;
}
uint64 sys_symlink(){
  char target[MAXPATH + 1], path[MAXPATH + 1];
  if(argstr(0, target, MAXPATH) < 0 || argstr(1, path, MAXPATH) < 0)
    return -1;
  return symlink(target, path);
}

symlink 시스템 콜은 심볼릭 링크를 생성하는 함수이다. 먼저 target과 path를 인자로 받아서 해당 경로에 심볼릭 링크를 생성한다. create 함수를 통해 심볼릭 링크를 생성하고, writei 함수를 통해 심볼릭 링크의 내용을 target으로 설정한다. create 함수에는 T_SYMLINK를 인자로 넣어 타입을 T_SYMLINK로 설정한다.

**결론**  
이번 프로젝트에서는 COW, Doubly Indirect Block, Symbolic Link를 구현했다. COW는 페이지를 공유하는 방식으로 메모리 사용을 최적화하는 기법으로, 페이지 폴트가 발생했을 때 페이지를 새롭게 할당하고 값을 복사하여 쓰기 작업을 수행하도록 했다. Doubly Indirect Block은 indirect 블록을 두 번 거쳐서 데이터를 저장하는 구조로, 파일 시스템에서 더 많은 데이터를 저장할 수 있도록 했다. Symbolic Link는 파일 시스템에서 다른 파일이나 디렉토리의 경로를 가리키는 특수한 파일로, symlink 시스템 콜을 통해 심볼릭 링크를 생성하고 해당 링크를 통해 파일에 접근할 수 있도록 했다. 모든 구현이 성공적으로 동작하며 테스트를 통과했다.
